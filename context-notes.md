# 컨텍스트 노트

## 2026-07-21
- 로컬 서버 검증 완료 (화면 + `api/judge.js` AI 판정 파이프라인 정상 동작 확인).
- 배포는 보류하기로 결정. 현재 버전은 프로토타입이라 수정할 부분이 많아서, 수정을 먼저 진행하고 배포는 나중으로 미룸.
- 수정 방향 3가지 큰 틀만 정함: 새로운 기능 추가 / 화면 디자인 / 게임 난이도 추가. 각 항목의 구체적인 내용은 아직 미정, 사용자가 추후 알려주기로 함.
- 사진 촬영 방식 결정: 캐주얼(기본) 모드는 PC 웹캠 있으면 실시간 캡처, 없으면 파일 선택 폴백 (접근성 우선). 랭킹 모드는 부정행위 방지를 위해 파일 선택 폴백 없이 라이브 캡처 강제 — 이유: 랭킹은 점수가 공개·비교되어 부정행위 유인이 커지지만, 캐주얼은 아직 랭킹/계정이 없어 부정행위 리스크가 낮음.
- DB 도입 시점 결정: 지금은 DB 없이 정적 사이트 + Vercel 서버리스 함수 구조를 유지 (무료, 관리 부담 없음). 랭킹 모드(계정 로그인 필요)를 만드는 시점에 DB를 붙이기로 함 — 이유: 랭킹모드 자체가 계정/점수 저장을 위해 DB를 필요로 하므로, 주제 중복 방지용 DB를 별도로 먼저 붙이는 것보다 랭킹모드 개발 시 한번에 도입하는 게 합리적. 무료 티어 한도 내에서 서비스하는 것을 우선한다는 방침도 이때 확인됨.
- 주제 중복 방지 방식 결정: DB 없는 현재 구조에서는 "셔플백(shuffle bag)" 방식 사용 — 난이도별 주제 풀을 10개보다 크게(예: 30~40개) 만들고, 날짜 인덱스를 풀 크기로 나눠 사이클 단위로 시드 셔플하여 풀 전체를 소진하기 전까지 반복이 안 나오게 함. 랭킹모드 DB 도입 시 "최근 N일 제외 랜덤 선택" 방식으로 이전 가능.
- 방침: 무료 이용 한도를 우선시함. 새 기능이 무료 티어를 넘어서거나 유료 서비스가 필요해지는 경우, 그때그때 사용자에게 알리기로 함.
- 타임어택 모드 결정: 주제는 AI 즉석 생성 대신 미리 만든 정적 풀 사용, 추후 계속 추가하는 방식 — 이유: 즉석 생성 시 API 왕복 시간이 "최대한 빨리 많이 깨기"라는 모드 핵심 재미를 해치고, 비용도 늘고, 품질(실제로 촬영 가능한 주제인지) 편차도 생김. 타이머는 실시간 기준 마감으로 결정 — 강제종료해도 시작 시각 기준으로 경과 시간 계산, 앱을 꺼놨다고 시간이 멈추지 않음.
- 사진 공유(갤러리) 기능 결정: 로그인은 랭킹모드 계정 시스템을 재사용하기로 함 — 이 기능만을 위해 별도 비로그인 방식을 먼저 만들면 나중에 계정과 다시 엮어야 하는 이중작업이 생기므로, 개발 순서상 랭킹모드(계정/DB) 이후로 미룸. 부적절한 사진(개인정보/선정적·유해)은 공지 문구만으론 강제력이 없어서, 기존 Claude API 판정 호출에 필터링 판별을 추가해 자동으로 걸러내기로 함. 저장 정책은 기본 판정용 사진은 저장 안 하고, 사용자가 명시적으로 "공유하기"를 누른 사진만 저장.
- 타이틀/모드선택 화면 구현 완료 (2026-07-21): 타이틀 → 모드 선택(기본모드만 활성, 카테고리/타임어택은 준비중) → 기본모드 진입 흐름. 로컬 서버(`http://localhost:3131`, 임시 dev-server.mjs)에서 정상 동작 확인 후 커밋 및 origin push 완료.
- 기본모드 난이도 선택 UI 구현 완료 (2026-07-21): 기존 계획이던 "난이도별 10단계"에서 "난이도별 7단계"로 변경됨(사용자 지시). 어려움 풀이 6개뿐이라 7개를 채우려 "이름이 영어로 적힌 물건" 1개 추가, 쉬움(10개)·보통(8개) 풀은 매일 시드 셔플 후 앞 7개만 사용. 난이도별 진행 상태는 `chalkak_easy`/`chalkak_medium`/`chalkak_hard` 키로 독립 저장 — 한 난이도 진행 중이어도 다른 난이도에 영향 없음. 이 리팩터링 과정에서 기존에 있던 테스트용 고정 주제("라면")가 자연스럽게 제거됨.
- 난이도별 실패 제한 구현 완료 (2026-07-21): 사용자 지시로 난이도마다 실패 기준을 다르게 함 — 쉬움은 제한 없음, 보통은 단계마다 3번 실패 시 종료(다음 단계 진입 시 카운트 리셋, `failScope: "stage"`), 어려움은 한 판 전체 누적 5번 실패 시 종료(`failScope: "run"`). DIFFS 객체에 `failLimit`/`failScope` 필드로 관리. 게임오버 시 `failScreen`에 "N단계에서 실패" 표시 + "난이도 다시 고르기" 버튼. 실패 카운트(`fails`)는 in-memory only — 새로고침/강제종료 시 초기화됨(진행 단계는 localStorage로 복원되지만 실패 횟수 유지는 요구사항에 없어 미구현). 모의 fetch로 4개 난이도 시나리오 검증 완료.
- 카테고리 모드 구현 완료 (2026-07-21): 카테고리 5개(회사/학교/집/카페/길거리), 각 11개 주제 풀(쉬운 오브젝트 + 어려움 모드 같은 창의적 주제 혼합 — 사용자 요청). 난이도 구분 없이 한 판에 랜덤 5개(당초 9개=난이도별3개 계획을 사용자가 "난이도 없이 랜덤 5개"로 변경). 날짜+카테고리 시드라 그날은 고정, 자정 초기화. 실패 제한 없음, 반복 플레이 가능(매번 1단계부터, localStorage 저장 안 함). 구현 시 게임 화면을 `run` 설정 객체 하나로 리팩터링 — `{ tierPrefix, label, failLimit, failScope, saveKey, selectScreen }`를 담아 기본/카테고리 모드가 같은 render/submit/save 로직을 공유. 기존 전역 `difficulty` 변수는 `run`으로 대체됨. 기본 모드 회귀 정상, 콘솔 에러 없음 확인.
- 사진 촬영 방식 구현 완료 (2026-07-21): "사진 찍기" 누르면 `getUserMedia`로 웹캠 시도 → 성공 시 프레임 안에 `<video>` 실시간 미리보기 + "촬영" 버튼, 실패(권한거부·미지원·카메라없음) 시 기존 `<input type=file capture=environment>`로 자동 폴백. "촬영" 누르면 canvas로 프레임 캡처(base64, maxSide 900)하고 스트림 즉시 종료. "다시 찍기"도 웹캠 재시도부터 시작. 모의 웹캠(canvas.captureStream) 주입으로 카메라/캡처/폴백 3경로 로컬 검증 완료. 주의: getUserMedia는 보안 컨텍스트(HTTPS 또는 localhost)에서만 동작하므로 배포 시 HTTPS 필수 — 실기기 테스트는 배포 후 확인 필요.
- 카테고리 모드 진행상황 표시 구현 완료 (2026-07-21): 사용자 지시로 카테고리별 문제 수를 5개→10개로 늘림(`CAT_STAGE_COUNT = 10`, 각 풀 11개라 충분). 주제 카드에 진행 배지(`0/10` 형식) 추가 — 통과한 개수/전체를 표시하고, 10개 다 통과하면 `CLEAR`(초록색) 표시. 배지는 카테고리 모드에서만 노출(`run.showProgress` 플래그로 게이팅, 기본 모드는 숨김). 통과 개수 계산은 render의 phase 기반 — "pass" 단계면 `stage`, 그 외엔 `stage-1`로 방금 통과한 걸 즉시 반영. Playwright 스모크 테스트로 0/10→증가→CLEAR, 기본모드 숨김, 카테고리 주제 10개 검증 완료.
- 카테고리 CLEAR 영구 저장 구현 완료 (2026-07-21, B안): 사용자가 "새로고침해도 클리어 정보 유지"를 원함 → 로그인/DB 없이 localStorage로 충분하다고 판단(같은 기기 범위). 두 방식(A: 오늘치 진행 유지/자정 초기화, B: 카테고리별 CLEAR 뱃지 날짜무관 영구보관) 중 B 선택. `chalkak_cat_cleared` 키에 `{ office: true, ... }` 형태로 클리어한 카테고리만 기록. 10개 전부 통과해 `stage > themes.length`가 되는 순간(render 진입부) `run.catId`로 markCatCleared 호출. 카테고리 선택 화면 진입 시(showScreen name==="cat"에 renderCatBadges 배치 → btnModeCategory/goBack/finScreen 뒤로가기 모든 경로 커버) 각 버튼의 `.clearbadge` span 토글로 CLEAR 뱃지 표시. 중간 진행상황은 저장 안 함(요구사항이 CLEAR 정보만) — 카테고리 재진입 시 여전히 1단계부터 시작하되, 한 번 클리어하면 뱃지는 계속 남음. 주의: 주제는 날짜 시드로 매일 바뀌지만 CLEAR 뱃지는 날짜와 무관하게 영구 유지(B안 의도). 다른 기기 간 공유는 랭킹모드 계정/DB 도입 시로 유지. Playwright로 클리어 전 미표시 → 클리어 후 해당 카테고리만 표시 → 새로고침 후 유지 검증 완료.
- 카테고리 중간 진행상황 저장 구현 완료 (2026-07-21): 사용자 요청으로 카테고리 모드에도 중간 진행 저장 추가. 별도 로직 없이 기본 모드가 쓰던 saveProgress/loadProgress를 재사용 — startCategory의 saveKey를 null에서 `chalkak_catprog_<cat>`로 부여하니 그대로 동작(loadProgress가 themes.length로 stage clamp, saveProgress는 btnSubmit 통과 시·btnNext 진입 시 호출됨). 진행은 날짜 단위 저장 — 주제가 날짜 시드로 매일 바뀌므로 같은 날은 이어서, 자정 지나면 saved.date 불일치로 stage 1 초기화(기본 모드와 동일). 카테고리별 독립 키라 회사/학교 진행이 서로 영향 없음. CLEAR 뱃지(`chalkak_cat_cleared`)는 별개 키라 그대로 날짜무관 영구 유지. loadProgress 주석의 "(카테고리)는 항상 처음부터" 문구가 이 변경으로 틀리게 되어 함께 정정. 완주(stage=11) 후 같은 날 재진입 시엔 저장된 stage=11이 복원돼 바로 fin(ALL CLEAR) 화면 — 의도된 동작(그날 주제는 이미 다 깼으므로). Playwright로 3단계 이탈→새로고침 재진입 이어하기, 카테고리 독립성, 자정 초기화, CLEAR 키 불간섭 검증 완료.
- 카테고리 모드 난이도 분리 + 고정 주제 개편 (2026-07-21): 사용자 요청으로 (1) 카테고리 선택 후 쉬움/어려움 난이도 선택 단계 추가, (2) CLEAR·진행을 (카테고리×난이도)별로 분리, (3) 날짜 무관 고정 주제로 전환. 화면 흐름: mode → cat(장소 선택) → catdiff(쉬움/어려움) → game. 각 카테고리에 easy/hard 10문제씩 고정 리스트 authoring(기존 11개 혼합 pool 제거, 날짜 시드 buildThemesForCategory도 제거하고 CATEGORIES[cat][diff].slice(0,10) 직접 반환). 저장 키: 진행은 `chalkak_catprog_<cat>_<diff>`, CLEAR는 `chalkak_cat_cleared`에 `<cat>_<diff>: true` 플랫 키. loadProgress에 `run.keepAcrossDays` 플래그 추가 — 카테고리는 true라 saved.date 체크를 건너뛰어 날짜 바뀌어도 진행/완료 유지(기본 모드는 여전히 날짜 리셋). CLEAR 뱃지는 catScreen에서 제거하고 catDiffScreen의 쉬움/어려움 버튼에 표시(renderCatDiffBadges, showScreen name==="catdiff"에서 갱신). selectedCat 전역 상태 추가. run 설정에서 catId→clearKey로 변경, tierPrefix에 카테고리명·label에 난이도명 넣어 게임 상단에 "회사 · 쉬움" 표시. Playwright로 흐름·주제 매칭·난이도별 CLEAR 분리·날짜변경 유지·카테고리 독립성·목록 길이(전부 10/10) 검증 완료.
- 도트 고양이 수집 보상 구현 완료 (2026-07-21): 사용자가 여러 스타일 비교 끝에 "처음 만든 굵은 도트" 스타일로 확정. 카테고리 어려움 클리어 시 카테고리별 고양이 1마리(색만 다름, 총 5마리: 회사 치즈냥/학교 회색냥/집 턱시도냥/카페 갈색냥/길거리 흰냥) 획득. 그림은 코드 생성 SVG(catSVG) — 16×16 매트릭스 도트, CSS로 바운스·꼬리 살랑·눈 깜빡임 애니메이션(.cat-anim 조상 클래스로 게이팅). 보상 소유 여부는 기존 CLEARED_KEY(`chalkak_cat_cleared`)의 `<cat>_hard` 값을 그대로 재사용(별도 저장 키 없음). markCatCleared가 신규 클리어 여부(boolean) 반환하도록 수정. 클리어 화면(finScreen)을 finReward/finDefault로 분기 — `_hard` 클리어면 고양이+이름+"새 친구 획득!"(이미 보유 시 "다시 만났어요!"), 그 외(쉬움·기본모드)는 기존 ALL CLEAR. 타이틀 화면에 "🐾 고양이 보상함" 버튼 추가, collection 화면 신설(획득=컬러 애니메이션 고양이, 미획득=어두운 실루엣+자물쇠).
